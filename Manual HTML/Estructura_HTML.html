Contenido básico de un documento HTML

Todo documento html contiene:

<!DOCTYPE html>
<html lang="es">

	<head>
		<meta charset="utf-8" />
		<title>Titulo de la página</title>
	</head>
	<body>

	</body>
</html>

La etiqueta head

Como ya he mencionado anteriormente, la parte head de una página Web no esta visible al usuario.La función de la cabecera es contener los metadatos del documento. En esta sección explicare los elementos principales que se pueden incluir en la cabecera de un documento HTML.

title

Utilización y características

Se utiliza para poner titulo a un documento. Tiene diversas utilidades:

  • Sera el texto que aparezca en la pestaña del navegador.
  • Sera el nombre que se sugiera para añadirla a marcadores, o favoritos, del navegador.
  • Es importante para los resultados de las búsquedas web. Lo utilizan como una de las referencias los buscadores web.

Declaración

titletitle

meta

Utilización y características

Empleamos esta etiqueta para definir la propiedades del documento HTML. Esta etiqueta recoge informacion acerca del autor, descripcion de la página, palabras clave, tipo de contenido...

Declaración

metameta

Atributos

  • name: Dependiendo del valor de este atributo estaremos describiendo una información u otra. Algunos ejemplos son:
    • author, que identifica al autor del documento HTML.
    • description,que describe el contenido del documento.
    • keywords, que define las palabras clave en el documento.
  • content: Especifica el valor del atributo name.
  • charset: Especifica el conjunto de caracteres que utilizara el documento.

Declaración

meta name="author" content="Nacho Salas"
meta name="description" content="Etiquetas Head"
meta name="keywords" content="HTML,HTML5,etiquetas,atributos,WEB"
meta charset="utf-8"

link

Utilización y características

Utilizaremos esta etiqueta para definir las rutas donde el documento HTML debe buscar hojas de estilos externas (CSS) o iconos.

Declaración

linklink

Atributos

  • href:Ruta donde se encuentra el recurso
  • rel:Relecion entre el docuemto HTML y el que estamos enlazando
  • type:Tipo de MIME. Es lo que utilizan los navegadores para reconocer el tipo de archivo, en lugar de la extensión. La lista es muy extensa, asi que aqui dejo un enlace: Lista MIME. Existen numerosos valores, pero las que nosotros vamos a utilizar mayoritariamente son:
    • "text/html": un archivo html (.html)
    • "text/css": un archivo CSS (.css)
  • media:Especifica el medio al que se aplica el recurso enlazado. Su valor ha de ser un media query.

Declaración

link href="estilo.css" rel="stylesheet" type="text/css"

base

Utilización y características

Sirve para dos cosas:

  • Proveer un URI "raiz" a partir del cual resolver todos los URIs relativos al documento.
  • Definir el comportamiento predeterminado en el que se abrirán todos los vínculos del documento.

Declaración

basebase

Atributos

  • href:Ruta raiz a partir de la cual se resolveran el resto de URIs
  • target:El nombre o palabra clave del contexto de navegación predeterminado, que establecerá cómo los vínculos del documento deben ser abiertos. Este valor puede ser:
    • "_blank": El documento enlazado se abrirá en una pestaña nueva
    • "_parent": El documento enlazado sea mostrado en el FRAMESET padre del frame actual.
    • "_self": El documento enlazado será mostrado en el mismo frame o ventana donde está el enlace.
    • "_top": El enlace sea mostrado usando todo el espacio de la ventana del navegador ignorando toda la estructura de frames. Este valor debe ser usado siempre que creemos un enlace a una página externa a nuestro sitio web.
    • atributo name de un iframe. De esto hablaremos mas adelante.

Declaración

base href="Aqui iria el URI raíz" target="_blank"
