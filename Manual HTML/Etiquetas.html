Etiquetas de sección

En esta seccion definiremos algunas de las etiquetas utilizadas para distribuir el texto en secciones o "cajas"

Para una lista completa de las etiquetas: TODOS LAS ETIQUETAS HTML

Tabla resumen

Etiqueta Utilización
header Encabezado de la página.
main Contenido importante, o principal, de la página.
footer Pie de página.
nav Sección con enlaces de navegación.
aside Contenidos relacionados, de manera lateral, con con el resto de la página.
section Sección cualquiera del documento. Como las secciones de un periodico (cultura, deportes, economía...).
article Define contenido autónomo dentro del documento. Como un artículo periodístico.
h1...h6 Titulos y subtitulos. Van desde h1 hasta h6. Cada uno es mas importante que el anterior.

Etiquetas de agrupación de contenido

En esta seccion definiremos algunas de las etiquetas utilizadas para agrupar los contenidos.

Para una lista completa de las etiquetas: TODOS LAS ETIQUETAS HTML

Tabla resumen

Etiqueta Utilización
p El texto se muestra como un párrafo.
pre Contenido preformateado.El formato se preserva.
blockcuote Contenido citado desde otra fuente.
figure Para insertar una imagen, grafico, diagrama...Su contenido es independiente del flujo principal de HTML.
figcaption Inserta una caja de texto en una imagen. Se anida dentro de figure.
div Contenedor genérico, sin significado. Es el contenedor mas "limpio" (no lleva estilo implicito).
br Salto de línea. Es una etiqueta vacia por lo que no necesita cierre.Se declara como br o como br/.
hr División de apartados. Coloca una linea horizontal, ademas de un salto de línea. Es una etiqueta vacia por lo que no necesita cierre.Se declara como hr o como hr/.

Etiquetas para listas

En esta seccion definiremos algunas de las etiquetas utilizadas para elaborar listas.

Para una lista completa de las etiquetas: TODOS LAS ETIQUETAS HTML

Tabla resumen

Etiqueta Utilización
ol Define una lista ordenada y numerada. Coloca numeros en orden a la izquierda de los elementos.
ul Define una lista desordenada. Coloca viñetas (bolos) a la izquierda de los elementos anidados dentro de ella.
li Cada elemento de una lista ordenada o desordenada. Se anidan dentro de ol o ul. Dentro de estos pueden anidarse ol y ul para crear listas anidadas.
dl Lista de definiciones. Es una lista de términos y sus definiciones asociadas.
dt Término a definir. Se definira por el siguiente dd. Si hay mas de uno se anidan dentro de dl.
dd Definición de un término. Define al anterior dt. Si hay mas de uno se anidan dentro de dl.

Atributos propios de las listas

Atributos ol

  • compact:Valor booleano, esta o no esta (se declara solo, sin = y sin "valor". Especifica que la lista es mas pequeña que lo normal. No se admite en HTML 5 (deberia utilizarse CSS).
  • reversed: El orden de la lista será en valores descendentes (9,8,7...).
  • start: Especifica en que número comenzará la lista. Como valor acepta un número.
  • type: Especifica el tipo de marcador que se utilizará en la lista. Como valores acepta:
    • "1"
    • "A"
    • "a"
    • "I"
    • "i"

Atributos ul

  • compact:Valor booleano, esta o no esta (se declara solo, sin = y sin "valor". Especifica que la lista es mas pequeña que lo normal. No se admite en HTML 5 (deberia utilizarse CSS).
  • type: Especifica el tipo de marcador que se utilizará en la lista. No se admite en HTML 5 (deberia utilizarse CSS). Como valores acepta:
    • "disc"
    • "square"
    • "circle"

Atributos li

  • value: Especifica en que número de este elemento en la lista. Como valor acepta un número. En listas ordenadas (ol), el valor se incrementará desde este número.
  • type: Especifica el tipo de marcador que se utilizará en la lista. No se admite en HTML 5 (deberia utilizarse CSS). Como valores acepta:
    • "1"
    • "A"
    • "a"
    • "I"
    • "i"
    • "disc"
    • "square"
    • "circle"

Ejemplos de listas

Declaración de Listas

                    <ol>
                        <li>Elemento 1</li>
                        <li>Elemento 2
                            <ul>
                                <li>Elemento 2 submenu</li>
                                <li>Elemento 2 submenu</li>
                            </ul>
                        </li>
                        <li>Elemento 3</li>
                    </ol>

Visualización de Listas

  1. Elemento 1
  2. Elemento 2
    • Elemento 2 submenu
    • Elemento 2 submenu
  3. Elemento 3

Declaración de Listas de Definiciones

                    <dl>
                        <dt>Palabra a definir: Difícil</dt>
                        <dd>Definición: Que presenta obstáculos</dd>
                        <dt>Palabra a definir: Fácil</dt>
                        <dd>Definición: Que no requiere gran esfuerzo, habilidad o capacidad</dd>
                    </dl>

Etiquetas semánticas de texto

En esta seccion definiremos algunas de las etiquetas utilizadas para la semantica de texto.

Para una lista completa de las etiquetas: TODOS LAS ETIQUETAS HTML

Tabla resumen

Etiqueta Utilización
a Representa un hiperenlace. Hablaremos de ellos mas adelante
em Etiqueta para un texto enfatizado.
strong Para resaltar partes importantes de un texto.
small Para realizar comentarios aparte (textos legales, notas explicativas...).
cite Marca una referencia a una fuente o el autor de un texto citado.
q Representa una cita textual (dentro de la linea). Puede contener el atributo propio, opcional, cite.
dfn Un termino cuya definicion se encuentra en la etiqueta padre más próxima.
abbr Representa una abreviación o un acronimo.
data Vincula el contenido de la etiqueta a una version del mismo legible por una máquina (programa). Esta version legible para la maquina la colocaremos en el atributo obligatorio value asociado.
time Sirve para introducir un periodo especifico de tiempo. Esto suele ser, una hora (formato 24 horas), una fecha del Calendario Gregoriano, o cualquier periodo de tiempo válido. Su contenido sera legible para la máquina si lo colocamos correctamente dentro del atributo opcional datetime.
code Representa un código informatico o de ordenador.
var Representa una variable(como expresion matemática o de programación) o similares (constantes, simbolos de física...).
samp Representa un output (salida) de un programa informático.
sub
sup		 Representan un subíndice y un superíndice respectivamente. Coloca un tipo de letra menor y ademas lo coloca más arriba o más abajo del centro de la línea. Es útil para representar un número elevado a una potencia (23) o para introducir marcas como el copyright (Nacho©).
i Coloca el texto en itálica (cursiva).
b Coloca el texto en nengrita.
u Muestra el texto subrayado.
s Muestra el texto tachado. Se utiliza para el texto que ya no es correcto.
mark Muestra el texto marcado (como con rotulador fluorescente).
bdi Texto que permite cambiar la direccionalidad del texto de manera aislada (el atributo asociado dir no se hereda). Sirve para introducir texto en idiomas que se leen en otras direcciones.
bdo Texto que permite cambiar la direccionalidad del texto de manera heredada (el atributo asociado dir se hereda). Sirve para introducir texto en idiomas que se leen en otras direcciones.
span Etiqueta de marcado sin un significado específico. Sirve para marcar contenido y poder atribuirle una class o un id, por ejemplo.
wbr Representa la oportunidad de un salto de linea. Donde se coloque la etiqueta indica los saltos de linea que se podran realizar. Se utiliza en texto sin espacios demasiado largo para una linea (URLs por ejemplo).

Etiquetas con caracteristicas especiales

q

Puede contener el atributo propio, opcional, cite. El valor de este atributo ha de ser una URL que referencia a la cita.

abbr

Puede contener el atributo general, opcional, title. La etiqueta otorga propiedades semánticas especiales a este atributo (como , por ejemplo, que no se hereda. El valor de este atributo solo ha de contener la definición del acronimo, nada más.

data

El atributo obligatorio value ha de tener como valor la traducción legible, para la máquina, del contenido de la etiqueta.

time

El atributo obligatorio value sirve para insertar una fecha u hora que una máquina pueda interpretar. Algunos valores válidos para este atributo son:

  • Cadena válida de año: 2020
  • Cadena válida de mes: 2020-02
  • Cadena válida de fecha: 2020-01-23
  • Cadena válida de fecha sin año: 02-23
  • Cadena válida de semana: 2020-W15
  • Cadenas válidas de hora: 15:03 || 15:03:24 || 15:03:24.315

bdi

Crea una diferencia semántica para el atributo genérico, opcional dir, ya que este no se hereda. Si este no esta definido, su valor por defecto es "auto", permitiendo al navegador decidir la dirección del texto basandose en el contexto del elemento.

bdo

Anula de forma explícita al algoritmo bidireccional Unicode. Hay que declarar el atributo dir con el valor "ltr", para la izquierda, o "rtl", para la derecha.
