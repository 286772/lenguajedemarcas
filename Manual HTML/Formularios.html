La etiqueta form

La etiqueta form es la etiqueta principal del formulario y en ella anidaremos el resto de las etiquetas.

Atributos

  • "action": Define la acción a realizar cuando se envia el formulario. Normalmente su valor es una URL donde se envía.
  • "method": Especifica el metodo por el que se enviara el formulario; sus valores son:
    • "get": Los datos son visibles en el campo de dirección de la página
    • "post": Se utiliza para enviar datos confidenciales
  • "enctype": Cómo se deben codificar los datos de formulario al enviarlos al servidor. Solo se admiten con el valor "post". Sus valores son:
    • "application/x-www-form-urlencoded":Valor por defecto. Todos los caracteres se codifican antes de enviarse (los espacios se convierten en símbolos "+" y los caracteres especiales se convierten en valores ASCII HEX)
    • "multipart/form-data": No hay caracteres codificados. Este valor es obligatorio cuando utiliza formularios que tienen un control de carga de archivos.
    • "text/plain": Los espacios se convierten en símbolos "+", pero no se codifican los caracteres especiales
  • "accept-charset": La codificacion de caracteres que se utilizará para el envío del formulario. Su valor ha de ser un conjunto de caracteres (ej: UTF-8).

La etiqueta label

Se utiliza para que el texto sirva como acción. Se añade a otras etiquetas y las complementa. Su atributo for ha tener como "valor" el valor del atributo id del elemento al que pretendemos enlazarla. Tambien puede tener un name.

La etiqueta input

Esta etiqueta crea un campo de entrada donde el usuario puede introducir datos. Puede contener los siguientes atributos generales (independientemente de su type.:

  • "id": Para poder relacionarla con un label.
  • "name": Pone nombre al imput
  • "value": Es un texto (excepto type concreto). Especifica el texto con el que comienza el input rellenado.
  • "required": Especifica si ese campo de entrada es obligatorio antes de enviar el formulario. Es un valor booleano.
  • "size": Es un número. Especifica el ancho visible, en caracteres, del un elemento.
  • "type": Especifica el tipo de dato que se va a introducir. Esto limita algunos de los atributos del elemento. Vamos a verlo más en detalle.

Tipos de input

Texto

type="text"

Sirve para introducir texto.

Atributos específicos

  • maxlength: Longitud máxima, en caracteres, que pueden introducirse.

Botón de Envío

Crea un botón para enviar el formulario.

type=submit

Atributos específicos

No tiene. Con value se le coloca un texto.

Valor numérico

Sirve para introducir números

type="number"

Atributos específicos

  • "min": Valor mínimo.
  • "max": Valor máximo.
  • "step": Intervalo entre números válidos. Por ejemplo, si el valor del atributo es 2, los numeros válidos serán: 0, 2, 4, 6...
  • "value": Ha de ser un número.

Casilla de verificación

Crea un elemento con una casilla para verificarlo.

type="checkbox"

Atributos específicos

  • "name": Aqui enumeraremos las diferentes input type="checkbox" como; ch1, ch2, ch3 ...
  • "value": Podemos darle un valor como texto. Tambien podemos marcarla cono "on" u "off" de manera predeterminada y nos dara igual como la marque el usuario.
  • "checked": La casilla comienza marcada. Valor booleano.

Boton de opción

Establece inputs de opciones, solo se puede elegir una.

type="radio"

Atributos específicos

  • "name": Todas las opciones han de tener el mismo nombre, asi se asocian.
  • "value": Aqui enumeraremos las diferentes input type="radio" como; ch1, ch2, ch3 ...
  • "checked": La casilla comienza marcada. Valor booleano.

Menús

Para crear menús utilizaremos la etiqueta select. Esta etiqueta tiene los atributos:

  • "name": nombre del menú.
  • "size": numero de opciones visibles del menú.
  • "multiple": especifica si es posible elegir mas de una opción.

Dentro de la etiqueta select anidaremos las etiquetas:

  • option: que serán las diferentes opciones. Tiene los siguientes atributos:
    • "value": Los valores de las opciones.
    • "selected": La que opcion que viene seleccionada por defecto. Es un valor booleano.

Tambien podemos agrupar estas opciones con la etiqueta optgroup para crear submenus. Admite el atributo label en el cual pondremos el criterio de agrupación. Se declara como un nivel intermedio entre select y option. Se anida en la primera y las option van anidadas dentro de ella.
